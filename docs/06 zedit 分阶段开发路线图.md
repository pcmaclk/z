# zedit 开发路线图（基于功能优先级）

## 第一阶段：MVP 核心 (v0.1.0)
**目标**：单文件编辑器，完全替代系统记事本
**时间预估**：4-6周

### ✅ 核心功能 (P0)
1. **文件基础操作**
   - 新建空白文件
   - 打开单个文件
   - 保存/另存为
   - 退出应用

2. **基础编辑功能**
   - 文本输入与删除（支持IME）
   - 复制、剪切、粘贴
   - 全选文本
   - 撤销/重做（10级深度）

3. **必要视图功能**
   - 显示行号
   - 状态栏显示行列位置
   - 字体设置（等宽字体）
   - Tab宽度设置 (2/4/8)

4. **编码基础支持**
   - UTF-8 (无BOM)
   - UTF-8 with BOM
   - 系统ANSI编码

### 🛠 技术实现重点
- Piece Table 核心缓冲区
- Slint UI 基础集成
- 基础事件处理流程
- 单光标+流式选区

## 开发原则（重申）

1. **严格遵守优先级**：P0 → P1 → P2 → P3
2. **每阶段可独立使用**：每个版本都是完整可用的编辑器
3. **不添加功能列表外的功能**：保持专注，避免范围蔓延
4. **性能优先**：始终保持大文件流畅编辑的能力
5. **架构不妥协**：坚持单向数据流和分层架构
